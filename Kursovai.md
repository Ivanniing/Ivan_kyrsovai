# 1. Анализ предметной области. Постановка задачи.
   
**1.1. Описание предметной области и функции решаемых задач**
Предметная область данной курсовой работы — это автоматизированная библиотечная система для средних специальных учебных заведений (ССУЗ). Основная цель системы — оптимизация и автоматизация процессов управления библиотечным фондом, а также упрощение взаимодействия между библиотекой и её пользователями (студентами и преподавателями).

Функции, решаемые в данной системе, включают:

- **Регистрация пользователей**: Создание учетных записей для студентов и преподавателей, что позволяет им получать доступ к библиотечным услугам.

- **Поиск книг**: Возможность поиска книг по различным критериям (название, автор, жанр) для удобства пользователей.

- **Выдача книг**: Регистрация факта выдачи книг пользователям с фиксацией даты выдачи и количества доступных экземпляров.

- **Возврат книг**: Обработка возврата книг, обновление статуса книги в системе и фиксация даты возврата.

- **Просмотр истории займов**: Позволяет пользователям видеть список книг, которые они брали, с указанием дат выдачи и возврата.

- **Управление библиотечным фондом**: Возможность библиотекарям добавлять, редактировать и удалять записи о книгах, а также отслеживать их доступность.

**5. Статистика по займам**: Общая информация о количестве выданных книг, активных займах и т.д. (для библиотекарей).

**2. Срок возврата**: Установленный срок, в течение которого книга должна быть возвращена (например, 2 недели).

# 3. Логическая структура БД

Логическая структура базы данных определяет, как данные будут организованы в базе данных, включая таблицы, их атрибуты, типы данных и связи между таблицами. В данной логической модели мы будем использовать реляционную модель данных.

**3.1. Определение таблиц и их атрибутов**

На основе концептуальной модели, определенной ранее, создадим таблицы для базы данных библиотеки для ССУЗ:

1. **Таблица 'Users' (Пользователи)**

- **'User ID'** INT PRIMARY KEY AUTO_INCREMENT — Уникальный идентификатор пользователя.
  
- **'FirstName'** VARCHAR(50) NOT NULL — Имя.
  
- **'LastName'** VARCHAR(50) NOT NULL — Фамилия.
  
- **'StudentID'** VARCHAR(20) NULL — Номер студенческого билета (для студентов).
  
- **'Email'** VARCHAR(100) NOT NULL — Электронная почта.
  
- **'User Type'** ENUM('Student', 'Teacher') NOT NULL — Тип пользователя.
  
2. **Таблица 'Books' (Книги)**

- **'BookID'** INT PRIMARY KEY AUTO_INCREMENT — Уникальный идентификатор книги.
  
- **'Title'** VARCHAR(255) NOT NULL — Название книги.
  
- **'AuthorID'** INT NOT NULL — Идентификатор автора (внешний ключ).
  
- **'GenreID'** INT NOT NULL — Идентификатор жанра (внешний ключ).
  
- **'PublicationYear'** YEAR NOT NULL — Год издания.
  
- **'AvailableCopies'** INT NOT NULL DEFAULT 0 — Количество доступных экземпляров.

3. **Таблица 'Loans' (Займы)**

- **'LoanID'** INT PRIMARY KEY AUTO_INCREMENT — Уникальный идентификатор займа.

- **'User ID'** INT NOT NULL — Идентификатор пользователя (внешний ключ).

- **'BookID'** INT NOT NULL — Идентификатор книги (внешний ключ).

- **'LoanDate'** DATE NOT NULL — Дата выдачи.

- **'ReturnDate'** DATE NULL — Дата возврата (может быть NULL, если книга еще не возвращена).

4. **Таблица 'Genres' (Жанры)**

- **'GenreID'** INT PRIMARY KEY AUTO_INCREMENT — Уникальный идентификатор жанра.
  
- **'GenreName'** VARCHAR(100) NOT NULL — Название жанра.
  
5. **Таблица 'Authors' (Авторы)**

- **'AuthorID'** INT PRIMARY KEY AUTO_INCREMENT — Уникальный идентификатор автора.
  
- **'AuthorName'** VARCHAR(100) NOT NULL — Имя автора.
  
6. **Таблица 'Librarians' (Библиотекари)**

- **'LibrarianID'** INT PRIMARY KEY AUTO_INCREMENT — Уникальный идентификатор библиотекаря.
  
- **'FirstName'** VARCHAR(50) NOT NULL — Имя.
  
- **'LastName'** VARCHAR(50) NOT NULL — Фамилия.
  
- **'Email'** VARCHAR(100) NOT NULL — Электронная почта.
  
**3.2. Определение связей между таблицами**

Связи между таблицами будут установлены через внешние ключи:

**1. Таблица Books (Книги)**

- AuthorID (FK) ссылается на Authors.AuthorID (PK)
Описание: Каждая книга написана одним автором, но один автор может написать несколько книг.

- GenreID (FK) ссылается на Genres.GenreID (PK)
Описание: Каждая книга принадлежит к одному жанру, но один жанр может включать множество книг.

**2. Таблица Authors (Авторы)**
   
- AuthorID (PK)
Описание: Уникальный идентификатор автора, на который ссылается поле AuthorID в таблице Books.

**3. Таблица Genres (Жанры)**
   
- GenreID (PK)
Описание: Уникальный идентификатор жанра, на который ссылается поле GenreID в таблице Books.

**4. Таблица Users (Пользователи)**
   
- User ID (PK)
Описание: Уникальный идентификатор пользователя (читателя), на который ссылается поле UserID в таблице Loans.

**5. Таблица Loans (Выдачи книг)**
   
- User ID (FK) ссылается на Users.UserID (PK)
Описание: Каждая выдача книги относится к одному пользователю, но один пользователь может взять несколько книг.

- BookID (FK) ссылается на Books.BookID (PK)
Описание: Каждая выдача относится к одной книге, но одна книга может быть выдана многим пользователям в разные времена.

**Итоговая структура связей**

Вот как будут выглядеть связи между таблицами:

**1. Books**

- AuthorID (FK) → Authors.AuthorID (PK)

- GenreID (FK) → Genres.GenreID (PK)

**2. Authors**

- AuthorID (PK) (связан с Books)

**3. Genres**

- GenreID (PK) (связан с Books)

**4. Users**

- User ID (PK) (связан с Loans)

**5. Loans**

- User ID (FK) → Users.UserID (PK)

- BookID (FK) → Books.BookID (PK)

**Заключение**

Эта структура связей между таблицами обеспечивает целостность данных и позволяет эффективно управлять информацией о книгах, авторах, жанрах и пользователях библиотеки. Внешние ключи помогают поддерживать связи между сущностями и гарантируют, что все записи будут корректными и согласованными.

# 4.  Физическая структура базы данных

Физическая структура базы данных (БД) описывает, как данные будут храниться на уровне хранения, включая типы данных, индексы, ограничения и другие аспекты, которые влияют на производительность и целостность данных. Ниже приведен пример физической структуры БД для библиотеки ССУЗ на основе ранее описанных сущностей.

**1. Таблица Authors (Авторы)**

- Описание: Хранит информацию об авторах книг. Поля CreatedAt и UpdatedAt позволяют отслеживать время создания и последнего обновления записи.

**2. Таблица Genres (Жанры)**

- Описание: Хранит информацию о жанрах книг. Поле Description может содержать дополнительную информацию о жанре.

**3. Таблица Books (Книги)**

- Описание: Хранит информацию о книгах. Поля CreatedAt и UpdatedAt позволяют отслеживать время создания и последнего обновления записи. Установлены действия при удалении (например, ON DELETE SET NULL), чтобы избежать потери данных.

**4. Таблица Users (Пользователи)**

- Описание: Хранит информацию о пользователях библиотеки. Поля Address и CreatedAt могут быть полезны для дальнейшей работы с данными.

**5. Таблица Loans (Выдачи книг)**

- Описание: Хранит информацию о выдаче книг пользователям. Поля CreatedAt и UpdatedAt позволяют отслеживать время создания и последнего обновления записи. Установлены действия при удалении (ON DELETE CASCADE), чтобы удалить все связанные записи при удалении пользователя или книги.

**Индексы**

Для повышения производительности базы данных можно добавить индексы на часто используемые поля: Authors, Genres, Books, Users, Loans.
